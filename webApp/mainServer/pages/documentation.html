1. Introduction

1.1. Purpose

GaMon will be an application that will offer an easy and simple way of managing the information related to the trash that is collected periodically in a given region. In this way, people will become more mature in the way in which they store trash and the environment in which they live will be cleaner.

1.2 Intended audience

This application will be designed to be used by the citizens of a certain region, by the persons that are responsible with the garbage collection and for the authorities that manage how the process of trash collection works at the large scale.

2. Overall description

2.1 Product functions

GaMon will have the following functionalities:

  • Generate either numerical or graphical reports organized in categories such as household waste, paper, plastic about the garbage that has been collected in some area. We will give this data in formats like HTML, CSV or PDF and we will cover different units of time (days, weeks, months).
  • Report the current situation regarding the collection of trash in a certain region, whether it is a clean zone or not.
  • See reports about the situation in the current zone.
  • Each waste container will have a QR code that will be scanned every time when a worker would want to collect trash.
  • Add or remove workers that collect trash.
  • Import/export data about the account of the workers in csv format.

2.2 Types of users

This application will support three groups of users: citizens, workers that collect trash and authorized people that manage the recycling process.

  • The citizens will act like the guests of the application because they that can only express their dissatisfaction and see the situation in their zone.
  • The workers will have an account that will offer information about what they have to do and their work will be monitored.
  • The persons that manage the whole process of trash collection will have another type of account and they will act like some administrators of the application. They will have the ability to see what each worker did and to compare that with what the citizens reported. Also, they will be the only users that will have the ability to create or delete accounts and to use the import/export functionality.

2.3 Operating environment

The users will access this application via a browser that can be hosted on both desktop operating systems like Windows, Linux or on mobile platforms such as Android, iOS. We will add an open source license to this project that will follow the terms stipulated by Creative Commons.

2.3.1 Frontend technologies

We will use HTML5, CSS3 and Javascript on the frontend side.

In order to simplify our work, we will use the Sass programming language because it lets us organize our css code in a better way. Also, our project will include the usage of Babel due to the fact that it offers us access to the modern functionalities of Javascript.

The application will have to be made responsive since it will be available on phone browsers.

2.3.2 Backend technologies

Node.js will be used to build the backend side because it is one of the best choices when it comes to execution speed.

No frameworks like Express will be used. Similarly, no SDK like googleapis-npm is allowed to be used. Everything will be made via a GraphQL/REST service with the help of libcurl library.

The MVC design pattern will be used to structure the application because it offers maintainability and it helps for parallel work. Besides this, the project will contain a personal an API GraphQL/REST. We will base our implementation on microservices and we will need to offer a minimal level of security that will protect our application against popular web attacks such as Cross Site Scripting sau SQL injection.

We will have to include an administration module that will be based on configuration files. Also, we will use Ajax in order to establish the communication between the client and the server.

Regarding how we will store data, we are going to use sqlite due to its simplicity.
